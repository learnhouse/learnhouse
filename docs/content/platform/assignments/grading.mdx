# Grading & Submissions

LearnHouse provides a straightforward grading workflow for teachers to review and score student submissions.

## Submission Flow

1. A teacher creates an assignment with one or more tasks (file upload or quiz).
2. Students complete each task and submit their work.
3. The teacher reviews submissions and assigns grades per task.
4. Grades are aggregated at the assignment level and reflected in the student's Trail.

## Grade Types

Each assignment can be graded using one of three scales:

- **Numeric** — raw number score (for example, 87 / 100).
- **Percentage** — score expressed as a percentage.
- **Alphabet** — letter grades (A, B, C, …).

## Grading Options

- **Task-level grading** — Assign a score to each individual task within an assignment.
- **Assignment-level grading** — Review and finalize the overall assignment grade.
- **Bulk review** — View all submissions for a task across students to compare and grade efficiently.

## Submission Status

Each submission moves through a clear set of states so teachers can see at a glance where every learner stands:

- **Not Submitted** — the student has not yet turned the task in.
- **Pending** — the student has saved work in progress.
- **Submitted** — the student has submitted and is awaiting a grade.
- **Late** — submitted after the deadline.
- **Graded** — the teacher has reviewed the submission and recorded a grade.

<Callout>
Submissions are tied to the student's Trail, so all grading data contributes to the overall course progress view.
</Callout>
