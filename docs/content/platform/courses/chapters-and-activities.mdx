# Chapters and Activities

Courses in LearnHouse follow a clear hierarchy: **Course → Chapters → Activities**. This structure keeps content organized and gives students a logical path through the material.

## Chapters

Chapters are the top-level groupings within a course. Use them to break your course into modules, units, or topics. You can reorder chapters at any time from the course editor.

## Activities

Activities are the individual pieces of content within a chapter. Each activity has a specific type that determines how content is presented to students.

### Dynamic Pages

Dynamic Pages use the built-in block-based editor to create rich, interactive content. You can combine text, images, code blocks, callouts, embeds, and more in a single page. This is the most flexible activity type.

### Videos

Video activities let you upload video files directly to LearnHouse or embed videos from YouTube and other external platforms. Uploaded videos are served through LearnHouse's media pipeline.

### Documents

Document activities display PDF files using a built-in viewer. Upload a PDF and students can read it directly within the platform without downloading.

### Assignments

Assignment activities are gradable tasks — currently file uploads and quiz-style submissions. Teachers can grade submissions from the dashboard, and results are recorded in the student's [Trail](/platform/courses/trails). See [Assignments](/platform/assignments) for details on task types and grading.

### SCORM

SCORM activities embed SCORM 1.2 or SCORM 2004 learning packages. This activity type is part of the Enterprise Edition.

### Custom

An escape hatch for custom activity implementations maintained outside of the built-in types.

<Callout>
Quizzes are available two ways: as an inline **Quiz** editor block inside a Dynamic Page (ungraded, lightweight knowledge checks) and as an **Assignment task** (gradable, stored submissions).
</Callout>

<Callout>
You can mix and match activity types within a single chapter to create a varied learning experience.
</Callout>
