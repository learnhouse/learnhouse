# SCORM e2e

End-to-end coverage of the EE SCORM feature: upload → analyze → import →
runtime tracking → resume → instructor reporting, plus a browser smoke of the
player.

## Requires Enterprise Edition

SCORM is an EE-only feature, so these specs need an **EE** instance (the default
`npx learnhouse@latest` self-host the suite boots is OSS and has no `/scorm`
routes). Point the suite at a running EE stack instead of booting one:

```bash
# From the repo root, in one terminal — start an isolated EE dev stack:
learnhouse dev --ee

# Then, against that instance:
cd apps/e2e
E2E_BASE_URL=http://localhost:3000 \
E2E_ADMIN_EMAIL=<admin email> E2E_ADMIN_PASSWORD=<admin password> \
bun run test features/scorm
```

The API specs (`01`, `02`) are pure REST and exercise the backend fixes
(de-dup import, no total_time double-count, valid 2004 tokens, resume,
write-only hygiene, authz, instructor results). The browser spec (`03`) drives
the player UI.

## Fixtures

`fixtures/*.zip` are generated by the synthetic corpus generator:

```bash
cd apps/api
uv run python -m src.tests.fixtures.scorm_packages ../apps/e2e/features/scorm/fixtures
```
